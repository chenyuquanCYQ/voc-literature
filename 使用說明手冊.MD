## 1. 系統需求

### 必要環境

| 項目 | 需求 | 備註 |
|------|------|------|
| Python | 3.10 以上 | `python3 --version` 確認 |
| Ollama | 已安裝並執行中 | https://ollama.com |
| Gemma3:12b | 已下載 | `ollama pull gemma3:12b` |
| Serper API Key | 建議取得 | 免費額度每月 2,500 次；無 Key 時自動改用 PubMed |
| 磁碟空間 | 至少 500 MB | 資料庫 + 模型快取 |
| RAM | 建議 16 GB | Gemma3:12b 需要約 8-10 GB |

### Serper API Key 取得方式

1. 前往 [https://serper.dev](https://serper.dev)
2. 免費註冊，取得 API Key
3. 填入 `config.yaml` 的 `serper_api_key` 欄位

> 💡 **無 Serper Key 也可運作**：系統會自動改用 PubMed E-utilities（完全免費），  
> 但搜尋廣度較小，建議有條件時取得 Serper Key。

## 2. 安裝步驟

### 方法 A：使用安裝腳本（推薦）

```bash
# 1. 進入專案目錄
cd voc_literature

# 2. 給予執行權限
chmod +x setup.sh

# 3. 執行安裝腳本
./setup.sh
```

### 方法 B：手動安裝

```bash
# 1. 建立並啟動虛擬環境
python3 -m venv venv
source venv/bin/activate          # macOS / Linux
# venv\Scripts\activate           # Windows

# 2. 安裝套件
pip install -r requirements.txt

# 3. 建立目錄
mkdir -p daily_reports exports

# 4. 初始化資料庫
python -c "from database import init_db; init_db('literature.db')"
```

### 確認 Ollama 狀態

```bash
# 啟動 Ollama 服務
ollama serve

# 另開終端，確認 gemma3:12b 已安裝
ollama list

# 若尚未下載，執行（約 8 GB，需等待）
ollama pull gemma3:12b
```

## 3. 設定說明

編輯 `config.yaml`，以下是各項目說明：

```yaml
# ─── 必填項目 ───────────────────────────
serper_api_key: "YOUR_SERPER_API_KEY_HERE"
# 填入你的 Serper API Key
# 若無 Key，保留原值，系統自動使用 PubMed

# ─── 搜尋關鍵字（可自訂增減）──────────
search_queries:
  - "exhaled breath metabolomics VOC biomarker"
  - "human volatile organic compounds disease"
  # 建議 10-20 個查詢，涵蓋你關注的子主題

# ─── 搜尋參數 ───────────────────────────
max_results_per_query: 10
# 每個關鍵字最多取幾篇，建議 5-15

# ─── 排程時間 ───────────────────────────
schedule_times:
  - "07:00"    # 早上執行一次
  - "19:00"    # 晚上執行一次

# ─── LLM 設定 ───────────────────────────
ollama_model: "gemma3:12b"
ollama_host: "http://localhost:11434"
llm_timeout: 120    # 每篇最長等待秒數，可依電腦速度調整

# ─── 評分門檻 ───────────────────────────
default_min_score: 2
# Dashboard 預設只顯示評分 >= 2 的文獻
```

### 自訂搜尋關鍵字建議

針對你的研究領域，建議加入：

```yaml
search_queries:
  # 你目前已有的主題
  - "exhaled VOC lung cancer clinical study"
  - "breath biomarker diabetes type 2"
  # 氣味醫學（你提到的子主題）
  - "olfactory medicine disease detection"
  - "odor-based diagnosis clinical"
  # 針對感測器
  - "MOX sensor VOC breath analysis"
  - "colorimetric sensor array exhaled"
  # 針對 AI
  - "neural network breath analysis classification"
```

```bash
0  7 * * * cd /你的路徑/voc_literature && /你的路徑/voc_literature/venv/bin/python scheduler.py >> logs/cron.log 2>&1

```

取得完整路徑：

```bash
cd voc_literature

which python                 # 取得虛擬環境 python 路徑（在啟動 venv 後）
```

建立 log 目錄：
```bash
mkdir -p logs
```

## 8. 文獻分類說明

### 類別定義

| 類別 ID | 中文名稱 | 說明 |
|---------|---------|------|
| `voc_breath` | 呼氣 VOC | 呼出氣體中的揮發性代謝物研究 |
| `voc_liquid` | 液態 VOC | 尿液、唾液、汗液的 VOC 採集分析 |
| `disease_cancer` | 癌症應用 | VOC 用於癌症診斷或篩查 |
| `disease_chronic` | 慢性病 | 糖尿病、腎病、肝病等慢性病 VOC 應用 |
| `infection` | 感染症 | 細菌/病毒感染的 VOC 生物標記 |
| `subhealth` | 亞健康 | 疲勞、壓力、睡眠的 VOC 監測 |
| `sensor_hardware` | 感測器 | 電子鼻、化學感測器硬體開發 |
| `ai_model` | AI模型 | ML/DL 應用於 VOC 數據分析 |
| `odor_medicine` | 氣味醫學 | 嗅覺診斷、氣味治療相關 |
| `method_tech` | 分析方法 | GC-MS、PTR-MS、SPME 等技術方法 |
| `other` | 其他 | 不相關或無法分類 |

### 評分標準（0-5）

| 分數 | 說明 | 典型情況 |
|------|------|---------|
| **5** | 核心相關 | 有完整人體VOC+疾病臨床試驗結果 |
| **4** | 高度相關 | VOC主題明確，有臨床數據或方法驗證 |
| **3** | 中度相關 | 涉及VOC/代謝體，但為方法或技術導向 |
| **2** | 邊緣相關 | 概念提及 VOC，或有潛在應用性 |
| **1** | 低度相關 | 僅在背景中提到，或間接相關 |
| **0** | 不相關 | 無關或誤判 |

### 手動調整分類

若 LLM 分類有誤差，可直接用 SQLite 工具修正：

```bash
# 使用命令行工具
sqlite3 literature.db

# 修改特定文獻分類
UPDATE literature SET category='voc_breath', score=4 WHERE id=123;

# 標記為相關
UPDATE literature SET is_relevant=1 WHERE id=456;

# 退出
.quit
```

或使用 [DB Browser for SQLite](https://sqlitebrowser.org/)（圖形介面工具，免費）。

## 9. 資料匯出與整合

### 匯出為 CSV

```bash
python scheduler.py --export-csv
# 輸出：exports/literature_export.csv
```

CSV 欄位：`ID, 標題, 作者, 年份, 期刊, 類別, 評分, 標籤, 一行摘要, DOI, URL, 來源, 加入時間`

### 匯入至 Excel / Google 試算表

1. 開啟 Excel → 資料 → 從文字/CSV 匯入
2. 確保編碼選擇 **UTF-8**（Windows 用 UTF-8-BOM）
3. 直接拖入 Google 試算表也可自動解析

### 整合現有 500 篇文獻

若你已有分類好的 CSV 檔案，可以批次匯入：

```python
# 在 Python 中執行
import pandas as pd
import sqlite3, json
from database import get_connection, init_db

init_db("literature.db")
conn = get_connection("literature.db")

# 讀取你的現有文獻 CSV
df = pd.read_csv("你的現有文獻.csv")

for _, row in df.iterrows():
    conn.execute("""
        INSERT OR IGNORE INTO literature
        (title, abstract, authors, year, journal, category, is_relevant, score, source)
        VALUES (?, ?, ?, ?, ?, ?, 1, ?, 'import')
    """, (
        row.get("title"), row.get("abstract"),
        row.get("authors"), row.get("year"),
        row.get("journal"), row.get("category", "other"),
        row.get("score", 3)
    ))

conn.commit()
conn.close()
print("匯入完成")
```

## 10. 故障排除

### ❌ Ollama 無法連線

```
[LLM] ❌ 無法連線 Ollama，請確認 `ollama serve` 已啟動
```

**解決方案**：
```bash
# 確認 Ollama 服務是否執行
ollama list

# 若未執行，啟動服務
ollama serve

# 確認 port 是否被占用
lsof -i :11434
```

---

### ❌ Serper API 錯誤 (401 / 403)

```
[Serper] 搜尋失敗: 401 Unauthorized
```

**解決方案**：
1. 確認 `config.yaml` 中的 `serper_api_key` 已正確填寫
2. 登入 serper.dev 確認 API Key 有效且有剩餘額度
3. 若無 Key，刪除或清空該欄位，系統自動改用 PubMed

---

### ❌ Gemma3:12b 不存在

```
Ollama 正常，但找不到 gemma3:12b
```

**解決方案**：
```bash
# 下載模型（約 8 GB，需要時間）
ollama pull gemma3:12b

# 或使用較小模型（修改 config.yaml）
ollama pull gemma3:4b
# config.yaml: ollama_model: "gemma3:4b"
```

---

### ❌ LLM 分類超時

```
[LLM] ⚠️ 超時 (120s)：Title...
```

**解決方案**：
1. 增加 `config.yaml` 中的 `llm_timeout: 180`
2. 若電腦效能不足，改用較小模型（gemma3:4b）
3. 關閉其他佔用 GPU/CPU 的程式

---

### ❌ 資料庫檔案不存在

```
sqlite3.OperationalError: no such table
```

**解決方案**：
```bash
python -c "from database import init_db; init_db('literature.db')"
```

---

### ❌ Streamlit 啟動失敗

```
ModuleNotFoundError: No module named 'streamlit'
```

**解決方案**：
```bash
# 確認虛擬環境已啟動
source venv/bin/activate

# 重新安裝
pip install streamlit
```

---

### ⚠️ 分類結果不準確

**可能原因與解法**：
1. **摘要太短**：Serper 回傳的 snippet 有時只有 1-2 句，可在 `config.yaml` 中加入 PubMed 搜尋補充完整摘要
2. **模型判斷偏差**：可在 `classifier.py` 的 `CLASSIFY_PROMPT` 中增加更具體的例子
3. **手動修正**：使用 DB Browser for SQLite 直接修改分類欄位

## 12. 常見問題 FAQ

**Q：每天會消耗多少 Serper API 額度？**  
A：以預設 15 個 queries × 每次 2 次/天 = 30 次/天。免費額度 2,500 次/月可用 80 天以上。

**Q：可以加入 arXiv 或其他來源嗎？**  
A：可以。在 `searcher.py` 中新增一個函數，參考 `search_pubmed()` 的格式，  
然後在 `run_all_searches()` 中呼叫即可。arXiv 有免費 API：`http://export.arxiv.org/api/query`

**Q：可以使用其他 LLM 模型（不用 Gemma3）嗎？**  
A：可以。在 `config.yaml` 修改 `ollama_model`，例如：  
- `llama3.2:3b`（速度較快）  
- `qwen2.5:14b`（中文理解較佳）  
- `mistral:7b`（英文效能好）

**Q：如何知道今天有沒有執行成功？**  
A：1. 查看 `daily_reports/` 是否有今天的報告  
2. `python scheduler.py --stats` 確認「今日新增」數字  
3. 查看 `logs/cron.log`（如有設定）

**Q：可以整合既有的 Zotero 文獻庫嗎？**  
A：可以從 Zotero 匯出 CSV，然後使用第 9 節的「整合現有文獻」方法批次匯入。

**Q：Dashboard 可以讓其他人存取嗎？**  
A：執行時加上 `--server.address 0.0.0.0`，區域網路內其他裝置可透過 IP 連入：  
`streamlit run dashboard.py --server.address 0.0.0.0`

**Q：LLM 分類每篇需要多久？**  
A：Gemma3:12b 在一般 16GB RAM 的電腦上，每篇約 15-45 秒。  
100 篇新文獻約需 30-75 分鐘。首次執行文獻多，後續每天通常只有 10-30 篇新文獻。
