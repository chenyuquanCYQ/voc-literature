四個分頁的用途與差異

Tab 1 · 總覽

5 個指標卡 + 類別分佈圖 + 搜尋記錄 + 最新高分文獻（score 4+）。不受側欄篩選影響，永遠顯示資料庫全局統計，適合每天開啟時快速掌握整體狀況。

Tab 2 · 文獻列表

受側欄所有篩選條件影響的完整表格，是日常查找文獻的主要工作區。

底部「詳細檢視」下拉選單只顯示當前篩選結果的前 50 筆。要找到更多可選文獻，把評分設為 0、時間選「全部」即可。選定後顯示完整摘要與原文連結。

Tab 3 · 統計分析

評分分佈、來源分佈（serper / pubmed / clinicaltrials / import）、每日新增趨勢、年份分佈、類別 x 平均分交叉表。適合每週回顧資料庫成長狀況與研究趨勢。

Tab 4 · 每日報告 vs 文獻列表的差異

每日報告是每次執行 scheduler.py 後自動生成的 Markdown 快照，只記錄當次新增的文獻，高分文獻優先排列並附完整摘要，適合直接分享給團隊成員。

文獻列表是資料庫全量查詢，可跨時間搜尋任何關鍵字，是自己研究時找文獻的工具。兩者互補，報告是推播通知，列表是搜尋工具。

常見操作情境

看全部 488 篇匯入文獻：時間選「全部」+ 評分 0 + 取消「只顯示相關」→ Tab 2
看今天新搜到的：時間選「今天」→ Tab 2 或 Tab 4 查報告
找肺癌相關文獻：關鍵字輸入「lung cancer」或類別選 disease_cancer
分享每日摘要給團隊：Tab 4 選日期 → 下載 Markdown → 貼到 Slack / Email
匯出給 Excel 使用者：側欄「匯出 CSV」→ exports/literature_export.csv
